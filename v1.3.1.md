# CraftDaemon v1.3.1

A **QoL polish patch** focused on deprecation cleanup, improved UX, and command discoverability.  
Main themes: (1) **Discord.js v15 preparation** - replaces deprecated `ephemeral`/`fetchReply` patterns with `MessageFlags`/`withResponse`; (2) **`/logs` rework** - subcommands instead of options, new `/logs stop`, configurable session timeout; (3) **`/help` command** - full command reference with categories, quick-start guide, and per-command detail pages; (4) **website extracted** to its own repository.

---

## ✨ Highlights

* 🧹 **Deprecation cleanup** - all 9 `ephemeral: true` → `flags: MessageFlags.Ephemeral`, all 3 `fetchReply` → `withResponse`. Silences Discord.js v14 warnings, zero breakage on v15.
* 📜 **`/logs` subcommands** - `live`, `tail`, and `stop` are now proper subcommands instead of a `mode` string option
* 🛑 **`/logs stop`** - users can now stop an active log session from Discord instead of waiting for the timeout
* ⏱️ **Configurable session timeout** - `LOG_SESSION_TIMEOUT_MS` env var replaces the hardcoded 60s (set `0` for no timeout)
* ❓ **`/help` command** - interactive embed with category-grouped command list and quick-start guide, switchable via dropdown
* 🔍 **`/help command:<name>`** - detailed per-command help with syntax, examples, permissions, and notes (with autocomplete)
* 🔐 **Explicit OWNER silent bypass** - `/exec silent` now correctly allows OWNER unconditionally instead of requiring `silentRoles` membership
* 🌐 **Website moved** to a separate repository at [d1vid3d.github.io/CraftDaemonSite](https://d1vid3d.github.io/CraftDaemonSite/)

## 🧹 Deprecation Fixes (Discord.js v15 Prep)

### Ephemeral → MessageFlags (9 occurrences)

Every `ephemeral: true` in reply/deferReply options has been replaced with `flags: MessageFlags.Ephemeral`:

| File | Context |
|---|---|
| `src/commands/logs.js` | Session guard, stop reply |
| `src/commands/exec.js` | Permission denied, silent restriction, blocked command |
| `src/events/interactionCreate.js` | Error handler fallback |
| `src/permissions/middleware.js` | Permission denied reply |
| `src/services/commandLock.js` | Lock busy reply |
| `src/commands/checkUpdate.js` | `deferReply` |

### fetchReply → withResponse (3 occurrences)

| File | Before | After |
|---|---|---|
| `src/commands/logs.js` | `reply()` then `fetchReply()` | `reply({ withResponse: true })` → `replyRes.resource.message` |
| `src/commands/ping.js` | `reply({ fetchReply: true })` | `reply({ withResponse: true })` → `replyRes.resource.message` |
| `src/services/execServices/confirmations.js` | `reply({ fetchReply: true })` | `reply({ withResponse: true })` → `replyRes.resource.message` |

### Import additions

`MessageFlags` was added to the `discord.js` import destructuring in all affected files.

## 📜 `/logs` Rework

### Subcommands instead of options

The old `/logs` used a `mode` string option (`live` / `tail`). Now uses proper Discord subcommands:

| Old | New |
|---|---|
| `/logs mode:live` | `/logs live` |
| `/logs mode:tail lines:10` | `/logs tail lines:10` |
| _not available_ | `/logs stop` |

### `/logs stop`

Stops the active live log session in the current channel:

- Shows `"⚠️ No active log session in this channel."` if none is running (ephemeral)
- Shows `"🛑 Log stream stopped."` on success (ephemeral)
- Internally calls `stopSession()` which kills the child process, clears the edit interval, and marks the Discord message with `--- [Stopped by user.] ---`

### Configurable session timeout

| Variable | Default | Description |
|---|---|---|
| `LOG_SESSION_TIMEOUT_MS` | `60000` | Auto-stop timeout in ms. `0` = no timeout (not recommended). |

The footer now dynamically reflects the configured timeout:
- `"Auto-stops after 60s"` (default)
- `"No timeout"` (when set to `0`)

### Buffer size increased

`MAX_LINES` in `logBuffer.js` bumped from **15 → 25** — more visible history in the live stream, still safely under Discord's 2000-char limit.

## ❓ `/help` Command (New)

A comprehensive help system with two interaction modes:

### `/help` (no options)

Shows an ephemeral embed with a **StringSelectMenu** to switch between two pages:

| Page | Content |
|---|---|
| **Commands Reference** | All commands grouped by category (Server Control, Admin & Tools, Information) with permission lock indicators |
| **Quick Start Guide** | Step-by-step getting-started guide (start → status → address → logs → exec → stop/restart) |

The select menu expires after **2 minutes** of inactivity and auto-disables.

### `/help command:<name>`

Shows detailed information for a specific command including:
- **Description** - what the command does
- **Usage** - exact syntax
- **Details** - in-depth explanation of behavior and edge cases
- **Examples** - usage examples
- **Required Permission** - the permission string needed
- **Notes** - important caveats

Supports **autocomplete** - start typing a command name and suggestions appear.

### Implementation

Data-driven architecture using `COMMAND_CATEGORIES` and `COMMAND_DETAILS` constants. Permission: `null` (public, no restriction). Follows existing embed patterns with `footer`, `timestamp`, and consistent colors.

## 🔐 Silent Mode OWNER Bypass

In `src/commands/exec.js`, the silent mode restriction now correctly exempts OWNER:

```js
// Before (bug): OWNER silently required silentRoles membership
if (silent && !permissionConfig.exec.silentRoles.includes(permResult.role))

// After (fix): OWNER bypasses unconditionally
if (silent && permResult.role !== "OWNER" && !permissionConfig.exec.silentRoles.includes(permResult.role))
```
