# Implementation Plan & Analysis
  
### 1. Deprecation Fixes (Discord v15 Preparation) `src/`
**Problem:** Discord.js v14 emits deprecation warnings for `ephemeral: true` and `fetchReply: true` / `interaction.fetchReply()`. These are removed in v15.

**Ephemeral → Flags pattern (9 occurrences):**

- `interaction.reply({ ..., ephemeral: true })` → `interaction.reply({ ..., flags: MessageFlags.Ephemeral })`

- `interaction.deferReply({ ephemeral: true })` → `interaction.deferReply({ flags: MessageFlags.Ephemeral })`

Files to update:

| File                              | Line       | Current                           | New                                             |
| --------------------------------- | ---------- | --------------------------------- | ----------------------------------------------- |
| `src/commands/logs.js`            | 68         | `ephemeral: true`                 | `flags: MessageFlags.Ephemeral`                 |
| `src/commands/exec.js`            | 72, 81, 97 | `ephemeral: true` (×3)            | `flags: MessageFlags.Ephemeral`                 |
| `src/events/interactionCreate.js` | 42         | `ephemeral: true`                 | `flags: MessageFlags.Ephemeral`                 |
| `src/permissions/middleware.js`   | 20         | `ephemeral: true`                 | `flags: MessageFlags.Ephemeral`                 |
| `src/services/commandLock.js`     | 30         | `ephemeral: true`                 | `flags: MessageFlags.Ephemeral`                 |
| `src/commands/checkUpdate.js`     | 14         | `deferReply({ ephemeral: true })` | `deferReply({ flags: MessageFlags.Ephemeral })` |

Also add `MessageFlags` to the import destructuring in each affected file:

```js

const { ..., MessageFlags } = require("discord.js");

```

**Fetch Reply → withResponse pattern (3 occurrences):**

| File                                         | Line  | Current                                                                 | New                                                                                                           |
| -------------------------------------------- | ----- | ----------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| `src/commands/logs.js`                       | 73-83 | `interaction.reply({...}); const msg = await interaction.fetchReply();` | `const replyRes = await interaction.reply({..., withResponse: true}); const msg = replyRes.resource.message;` |
| `src/commands/ping.js`                       | 13-19 | `fetchReply: true,` in reply options                                    | `withResponse: true` in reply options, access message from response                                           |
| `src/services/execServices/confirmations.js` | 43-58 | `fetchReply: true` in reply options                                     | `withResponse: true` in reply options                                                                         |

**Impact:** No visual/functional change. Silences v14 deprecation warnings; prevents breakage on v15.

### 2. /logs stop subcommand + Configurable Timeout
**Current state:** `sessionManager.js` has `stopSession()` but it is only called internally (on error, close, or 60s timeout). No command can trigger it. `SESSION_TIMEOUT_MS = 60000` is hardcoded.

**Changes needed:**
**a) `src/services/logsServices/sessionManager.js`:**
- Read `LOG_SESSION_TIMEOUT_MS` from env (default 60000 ms / 60s) instead of hardcoded constant.
- No other logic changes needed — `stopSession()` and `hasActiveSession()` already exist.

**b) `src/commands/logs.js`:**
- Add `"stop"` choice to the `mode` option (alongside existing `"live"` and `"tail"`).
- In the `execute()` function, add a handler for `mode === "stop"`:

  ```js

  if (mode === "stop") {

      if (!hasActiveSession(interaction.channelId)) {

          return interaction.reply({

              content: "⚠️ No active log session in this channel.",

              flags: MessageFlags.Ephemeral,

          });

      }

      stopSession(interaction.channelId, "Stopped by user.");

      return interaction.reply({

          content: "🛑 Log stream stopped.",

          flags: MessageFlags.Ephemeral,

      });

  }

  ```

- Import `stopSession` alongside `hasActiveSession` and `startSession`.

**c) `config/.env.example`:**
- Add new variable:
  ```

  LOG_SESSION_TIMEOUT_MS="60000"    # Default: 60000 | Suggested: 30000-300000 | 0 = no timeout

  ```

**d) `src/register-commands.js`:**
- No changes needed (auto-discovers command files).

**Impact:** Users can stop a running log session without waiting for the 60s timeout. Timeout is server-configurable.

### 3. /help Command (New File)

**Create `src/commands/help.js`:**
A comprehensive help command that provides:
- A list of all commands with descriptions grouped by category
- Permission info per command
- A quick-start or "getting started" guide

**Architecture:**
- Permission: `null` (public, no restriction).
- Use an embed with:
  - Fields grouping commands by category (e.g. Server Control: start, stop, restart, status; Admin: exec, logs; Info: ping, address, checkupdate, help).
  - A select menu (StringSelectMenu) to switch between "Commands Reference" and "Quick Start Guide" pages.
- Follow existing embed patterns (color, footer, timestamp).

**Data-driven approach:**

```js

const COMMAND_CATEGORIES = [

    {

        name: "Server Control",

        commands: [

            { name: "/start", desc: "Start the Minecraft server", perm: "server.start" },

            { name: "/stop", desc: "Stop the Minecraft server", perm: "server.stop" },

            { name: "/restart", desc: "Restart the Minecraft server", perm: "server.restart" },

            { name: "/status", desc: "Get server status, TPS, and player info", perm: "server.status" },

        ],

    },

    {

        name: "Admin & Tools",

        commands: [

            { name: "/exec", desc: "Execute Minecraft console commands via RCON", perm: "admin.exec" },

            { name: "/logs", desc: "View live or tail server logs", perm: "admin.logs" },

            { name: "/checkupdate", desc: "Check for bot updates", perm: "bot.checkUpdate" },

        ],

    },

    {

        name: "Information",

        commands: [

            { name: "/address", desc: "Get the server IP address", perm: "server.address" },

            { name: "/ping", desc: "Check bot latency", perm: null },

            { name: "/help", desc: "Show this help menu", perm: null },

        ],

    },

];

```

**Edge cases:**
- If a user doesn't have permission for a command, still show it but mark it with a lock icon 🔒 (or optionally filter by their permissions).
- Handle select menu timeouts gracefully (editReply to disable).
- Ensure ephemeral reply for privacy.
**Impact:** Provides discoverability for all commands, reduces support questions.
